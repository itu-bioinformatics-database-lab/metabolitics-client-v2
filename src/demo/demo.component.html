Creating an Account
Head to the top menu and click on ‘Login’ section, you will see 'Sign Up' text under the login button, click it and you will be redirected to the 'Sign Up' page to create a new account. As an other option, you may login as a demo user.



The user will have to fill the following form, then click the ‘Sign Up’ button, then the user will get a confirmation message for a successful registration, and can now login and proceed to use the website as a registered user.



Search

Search For Pathways/Metabolites/Reactions

  • In MetaboliticsDB, we can search for the following from the main page:
    • Pathways
    • Metabolites
    • Reactions
You can enter the first few characters and our autocomplete service will show you the names close to your search.



Search Metabolomics Analysis

You can find this subsection in the top menu 'Search' section. For better user experience, our search services support autocomplete for possible search texts based on the search characters you enter. In MetaboliticsDB we support searching public studies based on two search criteria:
  • Search studies based on Metabolites
    • Enter the metabolite name you wish to find studies that include it, then you will be provided with a list of studies that include it.
  • Search studies based on Pathways
    • Enter pathway name, change, diff quantifier, and diff amount and then press the 'Add' button and click 'Search' to get the results, you may choose multiple pathways to search at once.

Supported Data Formats
In MetaboliticsDB, we provided sample files for each format we support, they are provided in the 'Documentation' section. Metabolite names should be from one of our supported datasets:
  • ChEBI
  • HMDB
  • KEGG
  • PubChem

CSV Format:

First column entries should contain metabolite names and the second column entries should include their corresponding values/measurements.


JSON Format:

Keys should be metabolite names and the values should be their corresponding values/measurements.


Metabolitics Format:

In our format, we have two sheets:
  • Sheet one:
    • Sheet name should be “data”, it includes subjects ids/names and their corresponding metabolite names and values/measurements.
    • The first row (starting from the second column) should include subject ids/names.
    • The first column(starting from the second row) should include metabolite names from one of our supported datasets, each metabolite row entry should include the corresponding values/measurements for the subjects.

  • Sheet two:
    • Sheet name should be “meta”, it should include study name, group label, subject ids/names, and their corresponding labels.
    • The first row should include ‘study name’ as first entry and the study name of your choice as the second entry.
    • The second row should include ‘control/healthy/wildtype group’ in the first entry and its corresponding label in the second entry.
    • In the third row, the first entry should be ‘subject id’ and the second entry should be ‘group’ which will represent the subject label.
    • For the next columns (starting from row 4), first column should have the subject ids/names and second column should have the corresponding labels of the subject.

Metabolomics Workbench:

Please refer to the following link for a detailed tutorial on Metabolimics Workbench MWtab files format: https://www.metabolomicsworkbench.org/data/tutorials.php

Analyze
In the analyze section, we provided 3 options for the user to use our analysis service:
  • Uploading study files to be analyzed.
  • Manually adding metabolite changes to be analyzed.
  • Using a sample metabolite changes list.

Upload File

When choosing this option, you will be directed to the following page with various file formats that MetaboliticsDB support, for detailed information about the formats refer to 'Supported Data Formats' section.


Choose your preferred format, upload your study file, and you will be redirected to the following page:


Now, whether you choose to upload file, enter metabolites manually or sample metabolites list you will be directed to this page with your study data processed to be sent to be analyzed, but first, you have to provide the necessary information that maybe not be included in your uploaded file:
  • The user should fill the 'Disease / Physiological Condition' entry for the study.
  • The user will have to provide us with a study name.
  • The user will have the option to make his study public or private.
  • We also give the option to manually add or remove metabolites changes.
  • Last Step, the user has to choose the analysis algorithm, we currently support three analysis algorithms:
    • Metabolitics (our own analysis)
    • Direct Pathway Mapping
    • Pathway Enrichment Analysis
  • After clicking the 'Analyze' button, the user will be directed to the past analyses page.

Analysis Results Page

When we access an analysis result page, the analysis results are presented in multiple levels of details in both visual and tabular forms. A bar chart for the top 20 pathways with the highest absolute diff values is provided.



Below the bar plot, in a tabular form, all the pathways are listed with their computed diff values. Next to each pathway, two buttons are placed. The left button visualizes the corresponding pathway where each node represents a metabolite, and each edge represents a reaction. Edges are colored and thickened according to the corresponding reactions’ diff value. This form of visualization allows the user to quickly inspect what parts of the pathway experience the most change.


The right button, on the other hand,lists all the pathway reactions and their diff values in tabular form.



Comparison of Different Analysis Results for Different Datasets
MetaboliticsDB allows users to compare and contrast their metabolomics data analysis results to their previous analysis results, other users’ publicly available analysis results, and diseases and physiologicalconditions stored in the database. To achieve that, you have to access the 'Browse Analysis' subsection from the 'Browse' section in the top menu.


Next, the user will have to choose studies to compare. In order to compare between studies, the user will have to click the check buttons for all the studies he wishes to compare.


Next, the user will go up to the top menu and click the button 'Compare', in our case we choose two studies. Next we will be redirected to the comparison page. The comparison page features a heat map at the top where the rows represent pathways with the highest variance in terms of their diff values among the compared analysis results, and the columns represent the compared analysis results. Each cell in the heatmap is colored according to the corresponding pathway diff values. in the lower part of the page we will find each subjects pathways values.



Disease Association
  • Another novel feature of the MetaboliticsDB is that for each metabolomics analysis result, the closest set of diseases and physiological conditions are computed based on the similarity of the pathway-level metabolic behavior.
  • MetaboliticsDB stores metabolomics analysis results of a number of diseases (pre-computed and recorded in its database). Then, for each user submitted analysis run, the user is presented the top 5 diseases (e.g., diabetes) that have most similarity to the currently analyzed metabolomics data in terms of the activity level distribution over the metabolic pathways.
To check disease association, we access our analysis results page, then we will notice under the pathways bar plot there is a 'Similarity-based Disease Prediction' section, in it we will provide the top 5 similar cases and their similarity value to the analysis.




Disease Prediction
Disease prediction is an additional metabolomics data processing capability of MetaboliticsDB. Machine learning models are trained by MetaboliticsDB using past metabolomics data analysis findings for every disease on a regular basis. By saving the results as binary files, trained machine learning models can be used for prediction without the need for further training. In order to anticipate whether a certain disease is associated with the analysis or not, stored machine learning models are performed on the reaction diff values of future metabolomics data analysis findings.
